# GreyEd Brand Color System

## Overview

The GreyEd brand color palette is a sophisticated, professional color system designed for educational technology. The palette emphasizes readability, accessibility, and a modern aesthetic that appeals to educators, students, and families.

## Core Brand Colors

### Primary Colors

| Color | Hex Value | Tailwind Class | CSS Variable | Usage |
|-------|-----------|----------------|--------------|--------|
| **Navy** | `#212754` | `bg-greyed-navy` `text-greyed-navy` | `--color-greyed-navy` | Primary actions, headers, emphasis, teacher interface accents |
| **Blue** | `#bbd7eb` | `bg-greyed-blue` `text-greyed-blue` | `--color-greyed-blue` | Links, secondary actions, highlights, interactive elements |

## Opacity Variations

The greyed colors support Tailwind opacity modifiers for creating depth and hierarchy:

### Common Opacity Values

| Modifier | Opacity | Usage Example | Purpose |
|----------|---------|---------------|---------|
| `/10` | 10% | `bg-greyed-navy/10` | Subtle backgrounds, hover states |
| `/20` | 20% | `bg-greyed-blue/20` | Card borders, dividers, disabled states |
| `/50` | 50% | `border-greyed-blue/50` | Medium emphasis borders, overlays |
| `/80` | 80% | `text-white/80` | Secondary text on dark backgrounds |
| `/90` | 90% | `hover:bg-greyed-navy/90` | Hover states for solid colors |

### Examples in Use

```tsx
// Subtle hover background
<button className="hover:bg-greyed-navy/10">

// Soft border
<div className="border-2 border-greyed-blue/20">

// Secondary text on dark background
<span className="text-white/80">

// Active state
<div className="bg-greyed-navy hover:bg-greyed-navy/90">
```

#### Primary Action (Navy + White)

## Accessibility Guidelines

### Contrast Ratios

All greyed color combinations meet WCAG AA standards for text readability:

| Combination | Ratio | WCAG Level | Usage |
|-------------|-------|------------|--------|
| Navy (#212754) on White (#efeae4) | 8.5:1 | AAA | Large and small text |
| Black (#292828) on White (#efeae4) | 9.2:1 | AAA | All text sizes |
| White (#efeae4) on Navy (#212754) | 8.5:1 | AAA | Inverted text |
| Blue (#bbd7eb) on Navy (#212754) | 4.8:1 | AA | Large text only |
| Navy (#212754) on Beige (#dedbc2) | 7.1:1 | AAA | All text sizes |

### Best Practices

1. **Body Text:** Always use `text-greyed-black` or `text-black` for primary content
2. **Links:** Use `text-greyed-blue` with `hover:text-greyed-navy` for clear interactivity
3. **Backgrounds:** Prefer `bg-greyed-white` for main content areas, reserving pure white (#fff) for overlays
4. **Secondary Text:** Use `text-black/60` for less prominent information
5. **Borders:** Use opacity modifiers like `border-greyed-blue/20` for subtle separation

## Design Principles

### 1. Consistency First
Use greyed colors consistently across all interfaces to maintain brand cohesion.

### 2. Hierarchy Through Color
- Primary actions: `greyed-navy`
- Secondary actions: `greyed-blue`
- Tertiary actions: Borders or text only

### 3. Accessibility Always
Never compromise readability for aesthetics. Test contrast ratios before shipping.

### 4. Subtle Depth
Use opacity variations to create depth without introducing new colors.

### 5. Mobile-First Interactions
Ensure all interactive elements have appropriate hover, focus, and active states using the greyed palette.

## Migration Guidelines

### Replacing Legacy Colors

If you encounter these legacy colors in the codebase, consider migrating to greyed alternatives:

| Legacy Color | Greyed Alternative | Notes |
|-------------|-------------------|-------|
| `indigoBrand` | `greyed-navy` | Primary actions and headers |
| `mint` | `greyed-blue` | Links and secondary actions |
| Pure `white` (#fff) | `greyed-white` | Softer backgrounds |
| Pure `black` (#000) | `greyed-black` | More readable text |

### Migration Checklist

- [ ] Audit component for legacy color usage
- [ ] Replace color classes with greyed equivalents
- [ ] Test contrast ratios for accessibility
- [ ] Verify hover/focus states work correctly
- [ ] Test on mobile devices for tap highlights
- [ ] Update any inline styles using hex values
